Почему каждый WordPress SEO чек-лист, который вы читали, неполный

Откройте любой «WordPress SEO чек-лист», опубликованный до 2025 года, и найдёте ту же схему: 15–20 пунктов о мета-заголовках, мета-описаниях, alt-тегах изображений, XML-картах сайта, структуре ссылок и, возможно, основах разметки Schema. Это важно. Но это менее половины того, что поисковые системы — и ИИ-системы — оценивают в 2026 году.

Вот что традиционные чек-листы полностью упускают:

  • AEO-проверки: FAQ-схема (FAQPage), разметка speakable для голосовых ассистентов, схема SearchAction для поисковой строки sitelinks, схема автора/организации для сигналов достоверности, шаблоны прямых ответов, которые извлекают ИИ-ассистенты, оптимизация «Люди также спрашивают» и структурированные определения для голосового поиска.
  • GEO-проверки: Ясность сущностей и разрешение неоднозначности, структура контента, достойная цитирования, сигналы авторитетности, которые оценивают LLM, маркеры экспертизы, llms.txt для управления ИИ-краулерами, таблицы данных, цитируемые дословно, многоперспективный контент и разделы доверия с проверяемыми данными.

Часть 1: SEO-проверки (20 пунктов)

Это базовые проверки, которые должен пройти каждый WordPress-сайт. Вы наверняка знаете большинство — но даже опытные владельцы сайтов пропускают некоторые. Пройдите каждый пункт и проверьте его на своём сайте.

  • 1. Мета-заголовок присутствует и составляет 50–60 символов — каждая страница нуждается в уникальном, насыщенном ключевыми словами title
  • 2. Мета-описание присутствует и составляет 150–160 символов — убедительное описание с основным ключевым словом для кликов из SERP
  • 3. Канонический URL установлен правильно — самоссылающийся canonical на каждой странице против дублирования контента
  • 4. Тег H1 присутствует и соответствует намерению страницы — ровно один H1 на страницу с основным ключевым словом
  • 5. Иерархия заголовков логична (H1 → H2 → H3) — нет пропущенных уровней, заголовки вложены правильно для краулеров
  • 6. Разметка Schema валидна — минимум Organization, BreadcrumbList и схема типа страницы (Product, BlogPosting и т.д.)
  • 7. Open Graph теги заполнены — og:title, og:description, og:image, og:url и og:type для корректного отображения в соцсетях
  • 8. Twitter Card теги присутствуют — twitter:card, twitter:title, twitter:description и twitter:image для превью X/Twitter
  • 9. XML-карта сайта существует и отправлена — валидный sitemap.xml, указанный в robots.txt и отправленный в Google Search Console
  • 10. Robots.txt настроен правильно — разрешает сканирование важного контента, блокирует админ-пути, указывает карту сайта
  • 11. HTTPS применяется на всём сайте — нет предупреждений о смешанном контенте, все ресурсы по HTTPS
  • 12. Core Web Vitals пройдены — LCP менее 2,5с, FID/INP менее 200мс, CLS менее 0,1
  • 13. Alt-текст изображений на каждом изображении — описательные, релевантные ключевым словам alt-атрибуты
  • 14. Внутренняя перелинковка выстроена — каждая страница ссылается на 2–3 другие; нет осиротевших страниц
  • 15. Favicon установлен — иконка вкладки браузера для узнаваемости бренда в результатах поиска
  • 16. Мета-тег viewport присутствует — декларация адаптивного рендеринга для корректного отображения на всех устройствах
  • 17. Языковые и hreflang теги — атрибут html lang установлен; hreflang на многоязычных сайтах
  • 18. Нет битых внутренних ссылок — все внутренние ссылки возвращают статус 200; нет 404 в навигации
  • 19. Скорость страниц оптимизирована — скрипты отложены, шрифты предзагружены, изображения lazy-load, CSS минифицирован
  • 20. Заголовки безопасности настроены — HSTS, X-Content-Type-Options, X-Frame-Options, Referrer-Policy и CSP

Часть 2: AEO-проверки (15 пунктов) — Те, что большинство пропускает

Оптимизация для поисковых ответов — это про то, чтобы сделать ваш контент ответом, который извлекают ИИ-ассистенты, голосовой поиск и избранные сниппеты. Эти проверки гарантируют, что ваш WordPress-сайт структурирован для систем прямых ответов — избранные сниппеты Google, AI Overviews, Siri, Alexa и ChatGPT. Большинство WordPress-сайтов набирают менее 30/100 по AEO, потому что никто не рассказал им об этих проверках.

  • 21. FAQ-схема (FAQPage) на релевантных страницах — структурированные FAQ-разделы с правильной схемой Question/Answer для расширенных результатов Google и извлечения ИИ-ассистентами
  • 22. Разметка speakable объявлена — schema.org/speakable определяет, какие блоки контента подходят для озвучивания голосовыми ассистентами
  • 23. Схема SearchAction (WebSite) — активирует поисковую строку sitelinks в Google, сообщая о наличии функции поиска на сайте
  • 24. Схема автора с учётными данными — Person или Organization на каждой статье с именем, URL, областью экспертизы для сигналов E-E-A-T
  • 25. Параграфы прямого ответа — первый абзац под каждым H2 напрямую отвечает на заголовок как на вопрос в 40–60 словах для извлечения в сниппеты
  • 26. Определительные параграфы с шаблоном «это» — ключевые термины определяются шаблоном «[Термин] — это [определение]», который ИИ-системы извлекают как канонические определения
  • 27. Оптимизация «Люди также спрашивают» — выделенные H2/H3, соответствующие реальным запросам, с краткими ответами под каждым заголовком
  • 28. Готовность к голосовому поиску — контент содержит разговорные, вопросные заголовки и ответы на естественном языке
  • 29. Доступ ИИ-ботов разрешён — robots.txt не блокирует GPTBot, ClaudeBot, PerplexityBot и другие ИИ-краулеры
  • 30. Структурированный контент «как сделать» — пошаговые инструкции используют схему HowTo с нумерованными шагами и временем
  • 31. Блок резюме/TL;DR — заметно размещённый блок резюме, который ИИ-системы извлекают как быстрый ответ
  • 32. Разметка сравнительных таблиц — таблицы сравнения с чёткими заголовками, которые ИИ-поиск цитирует при вопросах «X vs Y»
  • 33. Схема хлебных крошек (BreadcrumbList) — разметка навигационного пути для понимания ИИ-системами иерархии сайта
  • 34. Семантические HTML5-элементы — правильное использование <article>, <section>, <aside>, <time> и <nav> для ИИ-парсеров
  • 35. Сигналы актуальности контента — видимые и машиночитаемые даты публикации и изменения с помощью <time datetime> и dateModified в схеме

Эти маленькие AEO-дополнения делают вас ответом в ИИ-поиске. Без них ваш контент существует, но никогда не выбирается. С ними голосовые ассистенты зачитывают ваш контент, сниппеты отображают ваш текст, а AI Overviews цитируют вашу страницу.

Часть 3: GEO-проверки (15 пунктов) — Новый рубеж

Оптимизация для генеративных систем — это про то, чтобы сделать контент достойным цитирования в ИИ-ответах. Когда ChatGPT, Perplexity или Google AI Overviews генерируют ответ, они берут из доверенных источников. GEO — путь стать таким источником. Это новейший рубеж оптимизации, и почти ни у кого нет для него чек-листа.

  • 36. Параграф-открытие с приоритетом сущности — первый абзац чётко указывает, о чём страница, кто её создал и почему она авторитетна — только ясная идентификация сущности
  • 37. Структура контента, достойная цитирования — контент организован так, чтобы отдельные параграфы можно было извлечь и процитировать независимо
  • 38. Сигналы авторитетности и маркеры экспертизы — квалификация автора, опыт, сертификаты и экспертиза явно указаны и машиночитаемы
  • 39. Файл llms.txt для управления ИИ-краулерами — файл, сообщающий ИИ-краулерам, о чём сайт, что приоритизировать и как атрибутировать цитаты
  • 40. Таблицы данных с проверяемой статистикой — таблицы с числами и данными, которые ИИ цитируют дословно при вопросах о данных
  • 41. Многоперспективный контент — контент, рассматривающий тему с разных сторон (за/против, варианты, альтернативы), демонстрирующий глубину для LLM
  • 42. Примеры кода и технические детали — конкретные примеры реализации и технические подробности для запросов разработчиков
  • 43. Раздел доверия с проверяемыми данными — раздел с сертификатами, наградами, числом клиентов и другими сигналами доверия
  • 44. Уникальные данные или оригинальные исследования — собственная статистика, результаты кейсов или бенчмарки, которые становятся магнитами цитирования
  • 45. Экспертные цитаты и атрибуция — именные цитаты экспертов с квалификацией, оцениваемые LLM как сигналы авторитетности
  • 46. Всесторонний охват темы — контент исчерпывающе охватывает тему с тематическим авторитетом через контентные кластеры и семантическую глубину
  • 47. Последовательное наименование сущностей — бренд, продукт и ключевые термины называются одинаково на всех страницах для графа сущностей ИИ
  • 48. Структурированный сравнительный контент — разделы «X vs Y» с чёткими сравнениями, которые ИИ извлекают при сравнительных вопросах
  • 49. Атрибуция источников и внешние цитаты — ссылки на авторитетные внешние источники, сигнализирующие глубину исследования
  • 50. Глубина длинного формата (1 500+ слов на ключевых страницах) — страницы с глубоким контентом ранжируются выше в ИИ-цитировании, так как LLM предпочитают всесторонние источники

Каждая GEO-оптимизация увеличивает вероятность цитирования ИИ. Страница с ясностью сущностей, таблицами данных, экспертными цитатами и структурой, достойной цитирования, экспоненциально чаще появляется в ответах ChatGPT или Perplexity.
